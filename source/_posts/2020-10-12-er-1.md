ER图笔记（一）

**ER图** Entity Relationship Diagram

### Cardinality
在线的端点通过下列符号来指定数量关系  
```
--  1
--||  1 && only 1
--o|  0 or 1
```

```puml
A-l-a
B-||b 
C-o|c
```
```
--{  multi
--|{  1 or more
--o{  0 or more
```
```puml
D-{d
E-|{e
F-o{f
```

### Direction
在线的中央插入下列关键字来指定方向
```
up, u
down, d, do
left, l, le
right, r, ri
```

```puml
A-u->Up
B-d->Down
```
```puml
C-l->Left
D-r->Right
```

### package style

可以使用自带的模版来定义每个包
```
package foo1 <<Node>> {
  class Class1
}
package foo2 <<Rectangle>> {
  class Class2
}
package foo3 <<Folder>> {
  class Class3
}
package foo4 <<Frame>> {
  class Class4
}
package foo5 <<Cloud>> {
  class Class5
}
package foo6 <<Database>> {
  class Class6
}
```
